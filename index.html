NapcoILockSDK

The Napco ILockSDK provides a comprehensive framework for connecting, controlling, and managing Napco smart locks using Bluetooth in your Android app. It handles credential decoding, lock operations, and status monitoring so you can focus on your app’s UI and business logic.

Overview

The NapcoILockSDK simplifies the integration of smart locks into your applications by providing a set of tools and APIs for connection, control, and interaction with locks. The SDK includes key components such as the Credential class for handling authentication data, ImportUtility class for importing and decoding lock credentials, and the ILock class for managing lock operations, and enums for lock states. With built-in support for delegate-driven communication, the SDK allows you to monitor lock status, RSSI levels, and handle operational mismatches effectively. Designed for flexibility and ease of use, the SDK can be integrated into a variety of smart home or access control systems.

Prerequisites

Before integrating the SDK, ensure you have:

  • Android Studio Ladybug or later with Kotlin 1.9+

  • Android Target API Level 31+

  • Napco developer account (to obtain an App ID)

  • Base64‑encoded lock credentials (exported from Napco systems through QR Code or provided via deep link)

Installation

Follow these steps to integrate the ILockSDK into your project:

Step 1: Add the Library

  1. Get the NapcoILockSDK-debug.aar or NapcoILockSDK-release.aar

  2. In your Android project:

    • Go to libs folder under the App folder

    • Place your .aar file into the libs folder

    • Add the dependency in your app level build.gradle file. For example:

API Structure

Types

  • ILockSDK.init(): Initialize the SDK with your Napco App ID, application context, and a callback to handle success or error events.

  • ImportUtility: Decode Base64 credentials into LockData models.

  • LockData: Model containing name, lockId, slot, password, and broadcastId.

  • ILock: Manage connection and lock operations for a specific lock.

  • LockStatusCallback: Interface for receiving callbacks on connection, unlock, errors, and RSSI updates.

Usage Overview

Decoding Credentials

Napco provides lock credentials as a Base64-encoded string. You can decode it into usable lock information like this:

val base64String = "<YOUR_BASE64_CREDENTIALS>"
val locks = ImportUtility.decodeCredentials(base64EncodedString = base64String)

for (lockData in locks) {
    println("Name: ${lockData.name}")
    println("Password: ${lockData.password}")
    println("Broadcast ID: ${lockData.broadcastId}")
}

Initializing the ILock SDK

Before using any SDK features, initialize it in your Application class inside the onCreate() method:

ILockSDK.init(applicationContext, appId) { result ->
    when (result) {
        Result.Success -> println("License has been loaded successfully")
        is Result.Error -> println("ILockSDK initialization failed. Error: ${result.error}")
    }
}

Unlocking or Toggling a Lock

Follow these steps to connect and perform lock operations using the SDK:

1. Initialize the Lock

Create an instance of ILock using the lock's broadcast ID (available via the broadcastId property of LockData).
Optionally, set an RSSI threshold to filter weak signals.

val lock = ILock(
    broadcastId = lockData.broadcastId,
    rssiThreshold = -50 // Optional
)

2. Create a Credential

Use the base64-encoded password from LockData to create a Credential.

val cred = Credential(fromBase64Password = lockData.password)

4. Wait for Connection

5. Unlock or Toggle the Lock

Once connected, perform the desired action using the credential:

Unlock:
